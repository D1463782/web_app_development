# 產品需求文件 (PRD) - 食譜收藏系統

## 1. 專案概述
- **背景與動機**：希望能有一個專屬的空間來記錄和收藏喜歡的食譜，方便未來查詢與製作。
- **目標用戶**：我自己（個人使用）。
- **核心價值主張**：提供一個簡單、直覺的介面，快速新增與查詢食譜，清楚列出所需的材料和料理步驟，提升料理時的效率。

## 2. 功能需求
作為系統的唯一使用者，我希望具備以下功能：

1. **新增食譜**
   - 作為使用者，我希望能夠新增食譜名稱與基本描述，以便記錄新的料理。
2. **新增與列出材料**
   - 作為使用者，我希望能在食譜中新增與清楚列出所需材料及份量，以便採買與備料。
3. **新增與列出步驟**
   - 作為使用者，我希望能在食譜中新增與列出料理步驟，以便在烹飪時能按圖索驥。
4. **瀏覽食譜列表**
   - 作為使用者，我希望能看到所有已收藏的食譜列表（可依建立時間反序排列），以便快速選擇和瀏覽我想做的料理。
5. **檢視與管理食譜（編輯/刪除）**
   - 作為使用者，我希望能查看單一食譜的完整資訊，並且能夠修改或刪除現有的食譜，以便保持收藏清單的正確性。

## 3. 非功能需求
- **技術限制**：預計採用 Flask (Python 網頁框架) + Jinja2 (模板引擎) + SQLite (輕量級資料庫) 進行開發。
- **效能與安全考量**：
  - 系統為個人使用，不需要承受高併發，但須確保操作流暢及資料存取穩定。
  - 初期可不實作複雜的會員認證與授權機制。
  - 建議支援響應式網頁設計 (RWD)，確保在手機或平板上開啟時，介面依然清晰，方便在廚房邊看邊做。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須要有 - MVP 核心)**
  - 瀏覽所有食譜的清單
  - 新增食譜（填寫名稱、材料清單、步驟清單）
  - 檢視單一食譜的詳細材料與步驟
- **Should Have (應該要有)**
  - 編輯已存在的食譜內容
  - 刪除不需要的食譜
- **Nice to Have (有的話會更好)**
  - 加入食譜圖片上傳與預覽
  - 幫食譜加上分類或標籤（如：甜點、主食、湯品）
  - 提供食譜搜尋功能（依關鍵字搜尋食譜名稱）
